## [v26.1.7](https://github.com/redpanda-data/redpanda-operator/releases/tag/charts%2Fredpanda%2Fv26.1.7) - 2026-06-30
### Changed
* Bump default Redpanda version to v26.1.12 release
### Fixed
* Boolean cluster properties set to `false` via `config.cluster` are no longer
silently overwritten by tiered storage defaults. Previously, setting a
property such as `cloud_storage_enable_remote_read: false` under
`config.cluster` had no effect because the merge that folds in tiered storage
defaults treated a boolean `false` as an empty value and let the default
(`true`) win. The rendered `.bootstrap.yaml` now honors an explicit `false`.
