## [v25.2.2](https://github.com/redpanda-data/redpanda-operator/releases/tag/operator%2Fv25.2.2) - 2026-03-16
### Changed
* Modified our statefulset liveness probes to just do a TCP check on the admin port. Previously it did an actual curl request that went through our internal service. This could potentially have some DNS-related issues and isn't really required to tell "hey is the admin API up".
### Fixed
* Allow Topic finalizer removal when broker is unreachable or credentials are missing during deletion, preventing stuck namespaces.
* In previous versions of the operator a field manager was unknowingly changed for resources that were synchronized via server-side apply. This can cause problems with modifying fields such as Service and StatefulSet port definitions. A post-upgrade migration job was added to remove any unwanted field managers.
* Fixed an issue where specifying pod affinity on a Redpanda cluster's `spec.clusterSpec.affinity` field resulted in rendering issues for the underlying StatefulSet.
